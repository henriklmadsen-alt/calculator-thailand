## Issue Details

### Root Cause
**Astro Configuration Mismatch:**
- Output mode: `hybrid` (no static prerendering)
- Sitemap generation: Expects static HTML in `dist/` directory
- Actual output: `dist/` contains only Astro's `client/` and `server/` build artifacts

### Evidence

**Sitemap URL Pattern (BROKEN):**
```
https://www.kamnuanlek.com/client/about/
https://www.kamnuanlek.com/client/หมวดหมู่/สินเชื่อ/
https://www.kamnuanlek.com/client/อัตราภาษีเงินได้2569/
```

**Expected URL Pattern:**
```
https://www.kamnuanlek.com/หมวดหมู่/สินเชื่อ/
https://www.kamnuanlek.com/อัตราภาษีเงินได้2569/
```

### Impact

- ❌ **Search Discovery**: Sitemaps cannot be submitted to Google Search Console
- ❌ **Indexing**: New/updated pages will not be discovered by search engines
- ❌ **Phase 1 Gate**: Blocks release (gate criteria requires 50 pages + 50 backlinks + 100+ organic users; cannot verify page discovery without working sitemaps)
- ❌ **SEO**: Trust signals damaged if broken sitemaps submitted

### Severity
**RELEASE-BLOCKING** — This is a critical SEO-facing issue that makes the site unsubmittable to search engines.

---

## CTO Architectural Decision Required

This is NOT a code bug. This requires a **structural decision** about how to reconcile Astro's hybrid mode with sitemap generation.

### Option 1: Static Prerendering
**Change**: `output: 'static'` in `astro.config.mjs`
- ✓ All 939 pages prerendered to HTML in `dist/`
- ✓ Sitemaps work immediately (existing script finds `index.html` files)
- ✓ Best for SEO (static content, fast delivery)
- ✓ Built-in sitemap functionality works
- ⚠️ **Trade-off**: Slower build time (estimate 90-120s for 939 pages), no dynamic server-side rendering

### Option 2: Dynamic Sitemap Generation
**Change**: Rewrite `generate-sitemap.mjs` to generate URLs without depending on `dist/` HTML files
- Approach 1: Parse `astro.config.mjs` routes and generate URLs programmatically
- Approach 2: Query page data from build manifest or metadata
- Approach 3: Make POST request to live server to enumerate routes
- ✓ Keeps hybrid mode (only prerender critical pages)
- ✓ Flexible, scales with new pages
- ⚠️ **Trade-off**: More complex, needs careful URL generation, must stay in sync with actual routes

### Option 3: Manual Sitemap
**Change**: Pre-build sitemap XML with all calculator URLs
- ✓ Works immediately
- ✓ No build script dependencies
- ⚠️ **Trade-off**: Manual maintenance, easy to miss pages when new calculators added
