### 3. Critical Regression: CAL-757 Property Transfer Redirect

**Issue Description:**
- Commit: `b0fd044 fix(cal-757): fix property transfer redirect template interpolation`
- Expected behavior: `/คำนวณค่าโอนบ้าน/` should serve HTML redirect with meta refresh to `/คำนวณค่าธรรมเนียมโอนบ้าน/`
- Actual behavior: Returns HTTP 200 but serves homepage HTML instead

**Source Code (as built):**
```astro
---
const targetUrl = '/คำนวณค่าธรรมเนียมโอนบ้าน/';
---
<!doctype html>
<html lang="th">
<head>
  <meta charset="utf-8" />
  <meta http-equiv="refresh" content={`0;url=${targetUrl}`} />
  <link rel="canonical" href={targetUrl} />
  <title>คำนวณค่าโอนบ้าน — เปลี่ยนเส้นทาง</title>
</head>
<body>
  <p>กำลังเปลี่ยนเส้นทางไปยัง <a href={targetUrl}>เครื่องคำนวณค่าธรรมเนียมโอนบ้าน</a></p>
</body>
</html>
```

**Live Response:**
- HTTP 200 with `Content-Type: text/html; charset=utf-8`
- Body: Homepage HTML (40+ calculator listing with full schema.org markup)
- NOT the redirect page expected

**Impact:**
- Users clicking property transfer calculator link get homepage instead of calculator
- Search engine crawling `/คำนวณค่าโอนบ้าน/` will index homepage content under wrong URL
- Trust damage: confusing user experience
- SEO damage: duplicate homepage content under different URL path

**Root Cause Hypothesis:**
- Build did not include new redirect page, OR
- Astro router has issue with Thai-character redirect routes, OR
- Railway/CDN caching issue with fresh deploy
